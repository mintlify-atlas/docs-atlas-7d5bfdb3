The `@magicyan/minecraft` package provides functions to fetch data from various Minecraft APIs with fully typed responses. It includes utilities for retrieving player skins, server status, and Mojang profile information.

## Installation

<CodeGroup>

```bash npm
npm install @magicyan/minecraft
```

```bash pnpm
pnpm add @magicyan/minecraft
```

```bash yarn
yarn add @magicyan/minecraft
```

</CodeGroup>

## Quick Start

### Fetch a Player's Skin

```typescript
import { fetchSkinRender } from '@magicyan/minecraft';

const result = await fetchSkinRender('Notch', 'default', 'full');

if (result.success) {
  console.log('Skin URL:', result.data.url);
  // Access the buffer: result.data.buffer
  // Access the arrayBuffer: result.data.arrayBuffer
}
```

### Check Server Status

```typescript
import { fetchServerStatus, isServerOnline } from '@magicyan/minecraft';

const status = await fetchServerStatus('mc.hypixel.net');

if (status.online) {
  console.log(`Server: ${status.version}`);
  console.log(`Players: ${status.players.online}/${status.players.max}`);
  console.log(`MOTD: ${status.motd.clean.join(' ')}`);
}

// Quick online check
const online = await isServerOnline('mc.hypixel.net');
console.log('Server is online:', online);
```

### Get Player Profile

```typescript
import { fetchMinecraftProfile } from '@magicyan/minecraft';

const result = await fetchMinecraftProfile('Notch');

if (result.success) {
  console.log('UUID:', result.data.id);
  console.log('Name:', result.data.name);
}
```

## Response Format

Most functions return a `FetchResult` type that provides a consistent error handling pattern:

```typescript
type FetchResult<T> = 
  | { success: true; data: T }
  | { success: false; error: string; code: number };
```

### Success Response

```typescript
if (result.success) {
  // TypeScript knows result.data is available
  console.log(result.data);
}
```

### Error Handling

```typescript
if (!result.success) {
  console.error(`Error ${result.code}: ${result.error}`);
}
```
